# ✦ Spark for iOS — local-first, on-device AI

The iOS version of Spark. Same idea as the web app — describe your criteria,
swipe on AI-generated activity ideas, and let the app learn your taste by
watching — but built around two principles:

1. **Everything runs and stays on the device.** Ideas are generated by the
   on-device Apple Intelligence foundation model (no API key, no network
   calls), and all data — every idea, swipe, and the inferred taste profile —
   lives in a local SwiftData store on the phone.
2. **Your insights are yours to take anywhere.** The *Your taste* tab exports
   a paste-ready markdown "taste dossier" (or full JSON) via the share sheet,
   so you can hand what Spark learned to any external LLM — ChatGPT, Claude,
   Gemini, or your own. Export is the only way data ever leaves the device.

## Requirements

- **Xcode 26** or later (the `FoundationModels` framework)
- **iOS 26** deployment target
- For real on-device generation: an **Apple Intelligence-capable device**
  (iPhone 15 Pro or later) with Apple Intelligence enabled
- On the simulator or unsupported devices the app falls back to canned sample
  ideas and a simple category-tally profile, so every screen is still testable

## Project setup

The Xcode project file is generated from `project.yml` with
[XcodeGen](https://github.com/yonaskolb/XcodeGen):

```bash
brew install xcodegen
cd ios
xcodegen generate
open Spark.xcodeproj
```

Alternatively, create a new blank SwiftUI iOS app in Xcode named `Spark`,
delete its template `ContentView.swift`/`SparkApp.swift`, and drag the
`Spark/` source folder into the project.

Then select your team under Signing & Capabilities and run.

## Architecture

```
Views (SwiftUI)          SwipeView · CardView · LikedView · ProfileView
     │
ViewModel                SwipeViewModel (@Observable, drives queue + learning)
     │
AI (protocol)            IdeaGenerating
     ├─ FoundationModelGenerator   Apple on-device model, guided generation
     └─ MockIdeaGenerator          canned pool + heuristic profile (fallback)
     │
Storage (SwiftData)      Idea · TasteProfile — local SQLite-backed store
     │
Export                   InsightExporter — markdown dossier / JSON, ShareLink
```

- **Generation** uses the `FoundationModels` guided-generation API
  (`@Generable` / `@Guide`), the on-device equivalent of the web app's JSON
  schema outputs — no fragile string parsing.
- **Learning**: every 4 decisive swipes, the recent like/dislike history is
  fed to the on-device model with a prompt asking it to infer the *why*
  behind the user's reactions. The resulting profile is stored locally and
  injected into every future generation request.
- **Prompt budget**: the on-device model has a modest context window, so
  history and already-seen titles are capped before prompting.
- The generator is behind the `IdeaGenerating` protocol — an MLX/llama.cpp
  backend or a remote API could be added without touching the UI.

## Privacy model

| Data | Where it lives | When it leaves |
|---|---|---|
| Criteria, ideas, swipes | SwiftData store on device | Never automatically |
| Taste profile | SwiftData store on device | Never automatically |
| AI prompts | Processed on-device by Apple Intelligence | Never |
| Exports | Generated on demand | Only via the user-initiated share sheet |

## Troubleshooting

**`SensitiveContentAnalysisML Code=15 … instruct_300m.safety: The data
couldn't be read because it is missing"` / `keyNotFound: 'thoughtContents'`**

These come from inside Apple's FoundationModels stack, not the app. They mean
the on-device model assets aren't actually present or are version-mismatched
on the machine running the app, even though the framework reported the model
as available. Fixes, in order of likelihood:

1. **Simulator:** Apple Intelligence in the simulator uses the *host Mac's*
   models. You need an Apple Silicon Mac running macOS 26 with Apple
   Intelligence turned on (System Settings → Apple Intelligence & Siri) and
   its model download fully finished. Intel Macs and macOS 15 hosts will fail
   exactly like this.
2. **Device:** Settings → Apple Intelligence & Siri → make sure it's enabled
   and the model download has completed (needs Wi-Fi, decent battery, and
   free storage; it can take a while after first enabling). Reboot afterwards.
3. **Version skew:** update to the latest iOS 26.x / macOS 26.x point release
   and matching Xcode — the `thoughtContents` decoding failure is the
   framework disagreeing with older on-disk model assets; asset re-download
   after an OS update clears it.

The app now also self-heals at runtime: if the on-device model throws during
generation, Spark switches to sample ideas with a banner instead of failing,
so you can keep testing the UX while the model assets sort themselves out.

## Status / known limitations

- Written without access to Xcode (authored in a Linux environment), so it
  hasn't been compiled yet — expect possibly a few small fix-ups on first
  build. Validate with: `xcodegen generate && xcodebuild -scheme Spark -destination 'platform=iOS Simulator,name=iPhone 16 Pro' build`
- Single taste profile (no multi-user)
- No "develop this idea into a plan" flow yet — natural next step, and pairs
  well with the export feature
- On phones without Apple Intelligence the app currently falls back to canned
  sample ideas. Planned: a real bundled/downloaded on-device LLM (llama.cpp +
  a small Qwen model) so every device gets intelligent generation — see
  [`LOCAL_MODEL_PLAN.md`](LOCAL_MODEL_PLAN.md).
